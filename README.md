# Aiyl Bank (Test Task)

Бэкенд-сервис для имитации банковских переводов между счетами. Проект демонстрирует построение отказоустойчивой бизнес-логики, безопасную обработку транзакций и профессиональное документирование API.

## Ключевые особенности

* **Идемпотентность:** Реализована проверка по `idempotencyKey`. Повторные запросы с тем же ключом не создают новых транзакций, а возвращают результат существующей, предотвращая двойное списание средств.
* **Защита от Deadlocks (Взаимных блокировок):** Использование **Pessimistic Write Locks** с принудительной сортировкой ресурсов (`order by accountNumber asc`). Это исключает зависание базы данных при встречных переводах (A → B и B → A одновременно).
* **Гарантированный Аудит:** Система фиксирует в БД каждую попытку перевода. При бизнес-ошибках (недостаток средств, неактивный счет) транзакция сохраняется со статусом `FAILED` и подробным описанием причины.
* **Валидация и безопасность:** Использование `Jakarta Validation` и строгих бизнес-правил (запрет селф-трансферов, проверка баланса, проверка статуса `ACTIVE`).
* **Профессиональная обработка ошибок:** Глобальный обработчик (`@RestControllerAdvice`) возвращает структурированные JSON-ответы с кодами 400/404/500 вместо стандартных серверных стектрейсов.



## Технологический стек

* **Java 17** (LTS)
* **Spring Boot 3.x** (Web, Data JPA, Validation)
* **PostgreSQL** (Основное хранилище)
* **Hibernate** (Pessimistic Locking)
* **Lombok** (Clean code)
* **Swagger/OpenAPI** (Интерактивная документация)

## API Документация

После запуска проекта документация доступна по адресу:  
`http://localhost:2020/swagger-ui/index.html`

### Основной эндпоинт:
`POST /api/transfers` — Создание перевода между счетами.

**Пример запроса:**
```json
{
  "fromAccountNumber": "123456789",
  "toAccountNumber": "987654321",
  "amount": 500.00,
  "idempotencyKey": "unique-uuid-v4-key"
}
```
## Инструкция по запуску

### 1. Предварительные требования
* Установленная **JDK 17** или выше.
* Запущенный экземпляр **PostgreSQL**.

### 2. Настройка базы данных
Вы можете настроить подключение через переменные окружения (рекомендуется) или напрямую в `src/main/resources/application.properties`:

properties
spring.datasource.url=${DB_URL:jdbc:postgresql://localhost:5432/aiylbank}
spring.datasource.username=${DB_USER:postgres}
spring.datasource.password=${DB_PASSWORD:1234}
